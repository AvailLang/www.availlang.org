Website Style Guide for AvailLang.org

Below are some examples of how to use the HTML tags and CSS styles in keeping with the style of our site. To copy these examples for use as code templates in a page you are making, view the html source of this page. Alternatively, view the source of a page whose style matches your new page.

Title for a Normal Content Page

The title for a page that is not editioned or time-sensitive appears alone at the top of the page in an h2 tag with style "content-title". Remember to open your paragraph tag (p) *after* the h2 to ensure proper spacing with page content.

Title for a Content Page that is Time-Sensitive and/or Serialized

Edition # - MonthName d, YYYY

Note the extra line between the metadata div and the text below, and that the open paragraph tag comes after the metadata. A page whose creation date is important should have just the date, while a revision date should say "Revised " first.

Section Heading within the Page

Heading h3 is for use to mark major topics within the content. Opening paragraph tags may come either before or after h3s.

Section Subheading

Heading h4 is for marking topics within an existing section marked by an h3. Anything smaller than h4 (h5+) is strongly discouraged. Consider the use of a list, as below.

Text that you wish to draw attention to may either be made bold by use of the strong tag, or italicized by use of the em (for empasis) tag. In tutorials, use strong to indicate a button, menu name, menu item, text field, radio button, etc. that the user should be looking at or using. Use em for text they should type in a text field.

When it is absolutely essential that a user see a section of text and understand it, use the paragraph style "text-highlight".

Utilize relative linking whenever you can, to minimize refactoring required if we move stuff around. For example, a link to a document in the same folder as the current page features a url that starts with "./". A link to a document in the parent folder starts with "../". For access to a file in another location but on the same domain as the current page should start with "/".

In text where you want to make an inline reference to Avail or other code elements like type name, variable name, or some other expression, use the code tag.

When linking to a page whose title is something normally in a code tag, use both. The order of their use does matter here. The code tag on the outside applies a font and a text color; the a tag overwrites the color with the site-wide link color.

When listing a file download inline with text, use the download-inline-icon class on the icon that corresponds to its filetype. See something.pdf or somethingelse.xls to see that these example links don't go anywhere.

When you are listing file downloads in an itemized list, use the download-list-icon style.

For code snippets that are a statement or longer, use two nested divs with "restricted-width-container" as the outer style, and "code-block" as one of the style elements on the inner style. This code block is of "neutral" code so its internal div class is "neutral-code code-block".

When you want to call attention to good programming style or code that works well, use "good-code code-block" as the inner style.

When calling attention to bad programming style or code that doesn't work as intended, use "bad-code code-block" as the inner style.

When simulating a command-line screen, use "console code-block" as the inner style.
