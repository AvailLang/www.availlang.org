Step 6: Import the Avail Preferences Set for Eclipse

The Avail preferences set will change the behavior of your Avail workspace. Changes immediately evident to the user include adjustments to Eclipse's default display behavior and a large set of code templates/text replacement settings that provide convenient access to Avail's many beautiful-but-keyboard-unfriendly Unicode characters. Less apparent are changes that affect the way the product is compiled, the run configurations that support the Avail builder, etc.

Installation of this preference set is required because these changes affect the consistency of experience and work product. Since these settings may differ drastically from a coder's personal preference or affect their ability to work on existing projects in a workspace, we recommend that Avail be installed in a fresh workspace. If you have imported Avail into an existing workspace but wish to retain its old settings, please delete the project, create and open a new workspace, and return to Step 4.

Once you are in the designated Avail workspace, complete the steps below.

Import the Avail Preferences File

  • If your Avail files may be out of date (i.e. if you did not just download Avail for the first time), do either a Fetch or a Pull, depending on your needs. (For more information on Git version control options, see Step 3 or http://git-scm.com/.)
  • From the File menu choose Import….
  • Expand the General folder and choose Preferences; hit Next.
  • On the next screen, the Import Preferences screen, hit Browse….
  • Navigate to the folder where your local copy of eclipse-juno.epf has been stored (this may vary if you have multiple branches). Select it, and hit Open.
  • The default setting is to Import all; if your Eclipse installation is dedicated to Avail development, you should do so. Otherwise, you can customize which preferences to apply by checking/unchecking boxes. Then hit Finish.
