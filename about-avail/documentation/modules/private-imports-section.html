Private Imports Section

The Uses keyword begins the private imports section. The contents of this section are zero or more import specifications separated by commas , (U+002C). An import specification is a declared dependency on a module, and it satisfies the ImportSpecification syntax diagram:

ImportSpecification

Where the terminal module is a string literal that names the import target. The import target is an upstream module by virtue of appearing in an import specification.

VersionList specifies a list of version requirements for the import target. It is a nonterminal that corresponds to the following syntax diagram:

VersionList

Where version is a string literal that denotes an acceptable API version of the import target. If the import target advertises no versions in common with the list of version requirements, then the compiler aborts compilation of the dependent module, i.e., the module whose header contains the import specification.

FilteredImportList specifies a cherry-picked list of exported names from the import target. It is a nonterminal that corresponds to the following syntax diagram:

FilteredImportList

Where the three terminals embedded in the syntax diagram are:

  • name: A string literal that specifies an atom that should be imported into the dependent module. Though it is sometimes useful to import an atom for its own sake, it is generally the case that the module wishes to gain access to a method named by the atom.
  • rename: A string literal that introduces a new atom that refers to the same method as the original atom.
  • excludedName: A string literal that denotes a name that is exported by the import target, but which should not be imported into the dependent module.
