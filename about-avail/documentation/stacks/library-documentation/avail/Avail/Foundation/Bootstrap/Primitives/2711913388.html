string, semantic restriction function
Source: /avail/Avail/Foundation/Bootstrap/Fallible Primitives
Categories: Primitives
Define the specified semantic restriction. Henceforth whenever the method named by methodName is invoked with arguments accepted by restriction , then apply restriction with the most specific available types of the actual arguments. This is determined by the compiler at the method's call site. A semantic restriction has two responsibilities. First, it may reject a method invocation on any appropriate semantic grounds by calling `Reject parse,expected:_`. Second, it may strengthen the type of value produced at the call site by answering a subtype of the method's declared return type.
