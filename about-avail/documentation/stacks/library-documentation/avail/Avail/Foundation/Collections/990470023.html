Construct and answer a stream that applies `predicate` to `aStream`. The result is lazy, so `aStream` may be infinite. The result contains only those elements of `aStream` that pass the specified filter. Note that the public stream operations may require an infinite amount of time to complete when presented with an infinite filtered stream, e.g., consider that case of: filter stream from 1 to ∞ by [ x : natural number | x < 10 ] Which will require infinite computation after advancing beyond the element `10`. Also, filtered streams are slow in general (because they must rescan for each public operation). Consider using filtered iterators instead.
