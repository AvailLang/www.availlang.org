reader, [⊥]→boolean
Source: /avail/Avail/Data Abstractions/Iterators/Filtered Readers
Categories: Data Abstractions, Readers
Construct and answer an reader that applies predicate to aReader. The result is lazy, so aReader may be infinite. The result contains only those elements of aReader that pass the specified filter.

Note that _is exhausted may require an infinite amount of time to complete when presented with an infinite filtered reader, e.g., consider the case of: 

 filter reader from 1 to ∞ by [ x: natural number | x < 10 ]

Which will require infinite computation after advancing beyond the element 10.

stream, [⊥]→boolean
Source: /avail/Avail/Data Abstractions/Streams/Abstract Streams
Categories: Data Abstractions, Streams
Construct and answer a stream that applies `predicate` to `aStream`. The result is lazy, so `aStream` may be infinite. The result contains only those elements of `aStream` that pass the specified filter.

Note that the public stream operations may require an infinite amount of time to complete when presented with an infinite filtered stream, e.g., consider that case of:

filter stream from 1 to ∞ by [ x: natural number | x < 10 ]

Which will require infinite computation after advancing beyond the element `10`.

Also, filtered streams are slow in general (because they must rescan for each public operation). Consider using filtered iterators instead.

Returns stream A stream capable of producing exactly those elements of `aStream` for which `predicate` answers `true`.
