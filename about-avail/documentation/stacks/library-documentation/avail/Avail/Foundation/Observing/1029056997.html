Enable Track variables that are read before written, then invoke traceFunction. For each variable dynamically reachable from traceFunction that was read before it is written, install a specialized write reactor. This write reactor will instigate a new trace of traceFunction, reinstall the specialized write reactor on the newly tracked variables, and compare the new result of traceFunction against its most recent successful result. If the results differ, then invoke action. Always invoke action after the initial trace.

This method is non-reentrant. If the method is reentered, then raise an illegal-trace-mode exception.

Position Name Type Description
Parameters
1 traceFunction []→any
2 action [⊥]→⊤ An arbitrary function that is invoked whenever traceFunction produces a new value. It is applied to the value produced by traceFunction.
Returns ⊤
Raises
exception If traceFunction raises an exception during the initial trace. Subsequently, traceFunction will be invoked by Observe_ as a consequence of evaluating the relevant write reactors.
illegal-trace-mode exception If the current fiber is already tracing (any mode).
