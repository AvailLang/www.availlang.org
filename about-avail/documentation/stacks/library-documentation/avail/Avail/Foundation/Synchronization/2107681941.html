Signal_

Definitions:

condition
Source: /avail/Avail/Foundation/Synchronization
Categories: Synchronization
Signal that the specified condition is satisfied. If any fiber is awaiting this signal, then schedule it to run and park the current fiber. A fiber blocked by this method is given preference over one blocked by Lock|lock_for_ during a monitor transfer. If no fibers are awaiting this signal, then do not block the current fiber; instead return control to the caller.
Position Name Type Description
Parameters
1 aCondition condition
Returns ⊤
Raises
cannot-signal-condition exception If the current fiber is not the owner of the condition's monitor.
condition-unsatisfied exception If the condition is not actually satisfied.
termination-requested exception If termination has been requested.
semaphore
Source: /avail/Avail/Foundation/Synchronization
Categories: Synchronization
Release a permit back to the specified semaphore. If any fibers are awaiting a permit, then schedule one to run and park the current fiber. A fiber blocked by this method is given preference over one blocked by Await_. If no fibers are awaiting the release of a permit, then do not block the current fiber; instead return control to the caller.
Position Name Type Description
Parameters
1 sem semaphore
Returns ⊤
Raises
cannot-exceed-maximum-permits exception If releasing a permit would exceed the maximum number of permits allowed by the specified semaphore.
termination-requested exception If termination has been requested.
