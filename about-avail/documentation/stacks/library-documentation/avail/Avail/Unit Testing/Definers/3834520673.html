Define a unit test.
Name Type Description
Parameters
name string The name of the test. This name will be resolved to an atom in the context of the current module. Since an atom retains its provenance, this permits the unit testing framework to report the module that defined the test. This also allows name collisions between modules.
id atom The test suite identifier of the test suite to which the new test should belong.
test []→⊤ The unit test.
optionalException <exception…|0..1> If specified, then test must raise an exception of this type when applied. If not specified, then test must not raise any exceptions.
Returns ⊤
Raises
test-already-exists exception If name names an existing unit test.
