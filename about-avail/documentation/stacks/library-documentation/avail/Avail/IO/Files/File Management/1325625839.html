Rename src to dest asynchronously. If the operation succeeds, then eventually apply succeed in the answered fiber. If the operation fails, then eventually apply fail to the error code in the answered fiber.
Position Name Type Description
Parameters
1 src file name The source file name.
2 dest file name The destination file name.
3 optionalReplace boolean true if an existing destination file should be replaced, false otherwise. If not specified, then default to false.
4 succeed []→⊤ What to do after the rename is successful.
5 fail [{ permission-denied code, file-exists code, no-file code, I/O-error code }ᵀ]→⊤, What to do with the error code if renaming fails. @param optionalPriority" byte? The priority at which to launch the answerd fiber. If not specified, then default to the priority of the current fiber.
Returns fiber→⊤ The fiber which will apply succeed or fail, as appropriate.
