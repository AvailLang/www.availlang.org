a fiber creating a directory named_«with file access rights_,⁇»then_,⁇else_,⁇«forked at priority_»

Definitions:

file name, {file access right|}?, []→⊤, [{I/O-error code, permission-denied code}}ᵀ]→⊤, byte?
Source: /avail/Avail/IO/Files/Primitives
Categories: Asynchronous IO, Files
Create a new directory asynchronously. If the operation succeeds, then eventually apply succeed in the answered fiber. If the operation fails, then eventually apply fail to the error code in the answered fiber.
Position Name Type Description
Parameters
1 fn file name The name of the target to create.
2 optionalRights {file access right|}? The file access right s with which to create the new directory. If not specified, then default to rw-r--r--. If the file system does not support file access right s, then just create the directory.
3 succeed []→⊤ What to do after the directory is successfully created.
4 fail [{I/O-error code, permission-denied code}}ᵀ]→⊤ What to do with the error code if directory creation fails.
5 optionalPriority byte? The priority at which to launch the answerd fiber. If not specified, then default to the priority of the current fiber.
Returns fiber→⊤ The fiber which will apply succeed or fail, as appropriate.
Raises
invalid-path exception If fn does not represent a valid path.
