Unlink fn . If no links remaining in the file system to the underlying file , then the file is deleted.
Name Type Description
Parameters
recursive boolean Unlink fn recursively.
fn file name
dontRequireExistence boolean true if fn need not name an existing file, false if it must.
followSymlinks boolean true if symbolic links should be followed during traversal, false otherwise. Applies to recursive unlinking only.
Returns ⊤
Raises
invalid-path exception If fn cannot be understood as a path.
partial-success exception If recursive is true and at least one file could not be unlinked for any reason.
permission-denied exception If permission is denied to use the specified file name .
no-file exception If dontRequireExistence is false and fn does not exist.
directory-not-empty exception If recursive is false and fn names a non-empty directory.
I/O exception If an I/O error occurs for any reason.
