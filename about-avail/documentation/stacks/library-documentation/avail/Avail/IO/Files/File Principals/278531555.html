Set the user principal that owns the file given by the specified file name.
Position Name Type Description
Parameters
1 fn file name
2 owner user principal
3 followIfSymlink boolean true if fn should be followed if it refers to a symbolic link, false otherwise.
Returns user principal
Raises
invalid-path exception If fn cannot be understood as a path.
permission-denied exception If permission is denied to use the file named by fn.
I/O exception If an I/O exception occurs.
operation-not-supported exception If user principal s are not supported by the host platform.
