Categories: IO, Networking, Sockets, IP
Asynchronously write data to a client socket. If the operation is successful, then fork a new fiber to apply success. If the operation is unsuccessful, then fork a new fiber to apply failure to the error code that classifies the failure. Answer the new fiber.
Position Name Type Description
Parameters
1 data byte* The bytes to write.
2 s client socket
3 success [byte*, boolean]→⊤ The function to apply if the write completes successfully.
4 failure [exception]→⊤ The function to apply to the exception that classifies the failure of the asynchronous write operation.
5 optionalPriority byte? If specified, then the priority of the new fiber. If not specified, then the new fiber will run at the same priority as the current fiber.
Returns fiber The new fiber which may eventually apply either success or failure.
