Asynchronously accept an incoming connection on the specified server socket. If the operation is successful, then fork a new fiber to apply success to the newly connected client socket. If the operation is unsuccessful, then fork a new fiber to apply failure to the exception that classifies the failure. Answer the new fiber.
Name Type Description
Parameters
s server socket
optionalName <nonempty string…|0..1> An optional string that describes the purpose of the client socket at a high level, provided for debugging purposes only. If not specified, then the default name «unnamed» is used.
success [client socket]→⊤ The function to apply to the newly connected client socket.
failure [exception]→⊤ The function to apply to the error code that classifies the failure of the asynchronous accept operation.
optionalPriority <byte…|0..1> If specified, then the priority of the new fiber. If not specified, then the new fiber will run at the same priority as the current fiber.
Returns fiber The new fiber which may eventually apply either success or failure.
